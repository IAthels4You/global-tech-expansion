# 3. IMPLEMENTATION PHASES (PRACTICAL GUIDE)
# ------------------------------------------

# PHASE I: Ingestion & Local Validation (Python)
# [ ] 1. Setup Environment: Ensure `.venv` is active and dependencies are installed. Create `.env` for secrets (`python-dotenv`).
# [ ] 2. Explore APIs: Make test requests to World Bank, UNESCO, and OECD APIs to analyze raw JSON structures.
# [ ] 3. Define Data Contracts: Based on exploration, define Pydantic schemas in `extraction/utils.py` (e.g., enforce ISO 3166-1 Alpha-3, data types).
# [ ] 4. Build Connectors (`extraction/connectors/`):
#     - World Bank: GDP PPP (NY.GDP.PCAP.PP.CD), Inflation (FP.CPI.TOTL.ZG), Population (SP.POP.TOTL), Broadband (IT.NET.BBND.P2).
#     - UNESCO: STEM Grads (FOSGP.5T8.F500600700), R&D % GDP (EXPGDP.TOT). *Header*: `Accept-Encoding: gzip`.
#     - OECD: Wages (DSD_EARNINGS@AV_AN_WAGE), Tax Wedge (DSD_TAX_WAGES_COMP@DF_TW_COMP). *Header*: `Accept: ...vnd.sdmx.data+json...`.
# [ ] 5. Clean & Format: Use Pandas/Numpy to structure the data locally and pass it through Pydantic validators.

# PHASE II: Cloud Infrastructure & Data Loading (Snowflake FinOps)
# [ ] 1. Provision Warehouse: Create X-Small warehouse with 60-second auto-suspend.
# [ ] 2. Provision Storage: Create database `GLOBAL_EXPANSION` and schemas `LANDING` (Bronze), `TRANSFORM` (Silver), `MART` (Gold).
# [ ] 3. Service Account: Setup roles, users, and permissions for Python and dbt access.
# [ ] 4. Bulk Load (Ingestion to Cloud): Use `snowflake-connector-python[pandas]` (`write_pandas`) to load validated DataFrames into the `LANDING` schema.

# PHASE III: Transformation & Governance (dbt Core)
# [ ] 1. dbt Initialization: Setup `dbt_project.yml` and configure connection to Snowflake (`profiles.yml`).
# [ ] 2. Source Definition: Define Snowflake `LANDING` tables as sources in dbt.
# [ ] 3. Snapshots (SCD Type 2): Implement `dbt snapshots` on raw risk/wage data to capture historical changes over time.
# [ ] 4. Staging Models (Bronze -> Silver): Clean, cast types, and standardize country codes (ISO 3166-1 Alpha-3) and education levels (ISCED).
# [ ] 5. Intermediate Models (Silver): Calculate complex business logic (Tax Wedge real cost, Cost Arbitrage, Inflation Volatility).
# [ ] 6. Governance: Add `dbt expectations` and generic tests (unique, not_null, accepted_values) to ensure data integrity.

# PHASE IV: Analytics & Visualization (Gold)
# [ ] 1. Build OBT (One Big Table): Create `fct_global_tech_matrix` joining Silver models to eliminate BI-layer joins.
# [ ] 2. Metric Calculation: Compute the "Weighted Score" directly in the OBT (40% Stability, 40% Talent, 20% Cost).
# [ ] 3. Data Catalog: Run `dbt docs generate` to document lineage and column descriptions.
# [ ] 4. Executive Dashboard: Connect BI tool to `MART.fct_global_tech_matrix` to visualize talent vs. cost dispersion quadrants.

# PHASE V: Predictive AI (Snowpark)
# [ ] 1. Setup Snowpark: Configure Snowflake environment to execute Python natively.
# [ ] 2. Time-Series Forecasting: Implement `Prophet` to predict Economic Stability and STEM grads 10-20 years ahead.
# [ ] 3. Clustering: Implement `scikit-learn` (K-Means/Agglomerative) to group emerging talent hubs based on growth trajectories.
# [ ] 4. CI/CD Automation: Finalize GitHub Actions workflows to automate the entire ELT and ML pipeline.
